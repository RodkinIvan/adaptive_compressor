Adaptive Compressor

A byte-level hierarchical language model with causal adaptive borders. The model climbs from bytes to rarer meta-tokens only when cumulative uncertainty within the current span becomes large enough, then decodes back down to bytes with causal span conditioning.

Figure A. Full Model

Encoder levels compress the sequence by promoting border states upward. Decoder levels mirror that hierarchy and feed higher-level context back down, but only to positions after the corresponding border.

Figure A summarizes the current architecture: byte encoder on the left, recursive meta encoders in the middle, and mirrored causal decoders on the right.

Figure B. Cumulative Border Rule

Routing in default uncertainty mode is now simple and streamable: keep a running uncertainty sum within the current span, emit a border when the sum crosses a constant threshold, then reset to zero.

The same rule is used at higher levels, except the per-step quantity is predicted meta-uncertainty instead of byte entropy.

Figure C. Exact Streaming Inference

Cached inference now matches full prefix recomputation. The scheduler is event-driven: low level runs every byte, higher levels run only when a new border is emitted below.

The cached scheduler is now exact: step-by-step cached inference matches full prefix recomputation up to floating-point noise.

Talking Points

Bytes all the way down: the model never relies on an external tokenizer. Higher levels are created internally by promoting uncertain boundary states.
Why cumulative uncertainty? Instant uncertainty can stay modest forever. Accumulating it over a span makes borders inevitable for sufficiently hard local regions.
Why the mirrored decoder? Higher levels are not only compressed summaries; they are fed back down causally so the byte predictor can use abstract context.
Fair comparison: use the parameter-matched baseline and compare byte-level LM metrics such as byte_encoder_bpb and byte_decoder_bpb carefully.
